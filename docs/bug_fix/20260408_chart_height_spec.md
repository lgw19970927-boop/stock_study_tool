## 一、常數定義

| 常數名稱                  | 值      | 說明                                                     |
|---------------------------|---------|----------------------------------------------------------|
| `_paneHeightOverhead`     | 52 px   | LightweightCharts 內部保留空間（scrollbar／時間軸等），由 LW 自動處理；**不**用於各 pane setHeight 計算 |
| `_defaultSubPaneHeight`   | 120 px  | 新增副圖時的基準高度（實際初始高度 = `120 × (containerHeight / 420)`，見下方說明） |
| `_minMainPaneHeight`      | 60 px   | 主圖 pane 最小高度                                        |
| `_minSubPaneHeight`       | 60 px   | 副圖 pane 最小高度                                        |
| `_minTotalChartHeight`    | **420 px** | 整體容器最小高度（2026-04-09 由 300px 提升）；使用者「只能拉大，不能比預設更小」 |

> **各 pane setHeight 目標值之總和 = totalContainerHeight**（LW 直接接受總高，無需手動扣除 overhead）  
> **主圖高度** = totalContainerHeight − Σ(副圖 savedHeight)（主圖永遠填滿剩餘空間）  
> **新增副圖初始高度** = `120 × (containerHeight / 420)`，而非固定 120px（在 420px 容器下 = 120px；在 840px 容器下 = 240px）  
> （`_paneHeightOverhead = 52px` 保留為常數，僅用於 `_getMinTotalHeightForState` 容器最小值估算，**不**參與各 pane setHeight 計算）

## 二、六條核心規則（2026-04-08 確認版）

> 以下規則為圖表高度管理的唯一權威規格，取代所有先前版本的設計假設。

1. **整體圖表容器一定處於填滿狀態**：無副圖時主圖填滿容器；有副圖時主圖＋副圖合計填滿容器。
2. **整體圖表容器放大縮小後**，主副圖會依放大縮小前的佔比進行縮放（等比縮放）。
3. **新增副圖的空間源自主圖的壓縮**（固定高度副圖），不從其他副圖借空間。新增的每個副圖高度一致（`_defaultSubPaneHeight = 120px`）。移除副圖時，歸還空間給主圖。
4. **拖拉主圖與副圖的分隔線、副圖與副圖的分隔線**會改變各 pane 的佔比，此佔比在後續容器放大縮小時被保留。
5. **頁面重新整理時**，整體容器高度為整體容器最小高度（`_minTotalChartHeight = 300px`）。
6. **整體圖表容器大小僅能透過「拖拉調整圖表整體高度」來調整**，內部主圖、副圖不能任意撐開整體圖表容器的高度。

> 不自動擴展容器，改為**強制關閉最先開啟的副圖**（等同點擊 ✕ / 圖表管理取消勾選）。

```
 【觸發條件】

    ‧ 新 main = 60 + 120 − 120 = 60px ≥ 60px ✅

 ┌──────────────────────────┐ ─┐      容器維持 300px（不擴展！）
 │  主圖   mainH = 60px     │ ─┘      VOL 已被強制關閉，等同手動點擊 VOL ✕ 或取消勾選
 ╠══════ ↕ 分隔線 ══════════╣
 │ ⚙ ✕  RSI    savedH=120  │
 ├──────────────────────────┤
 │  RSI 繪圖                │
 ╠══════ ↕ 分隔線 ══════════╣
 │ ⚙ ✕  MACD   savedH=120  │ ← 成功新增
 ├──────────────────────────┤
 │  MACD 繪圖               │
 └──────────────────────────┘
 ≡≡≡≡ 底部 handle（容器未擴展）≡≡≡≡
```

> **注意**：
> - 若強制關閉一個副圖後主圖仍 < 60px，繼續關閉下一個最先開啟的副圖（cascade）
> - UI 同步：強制關閉等同手動關閉，圖表管理中的對應項目同步取消勾選
> - 「最先開啟」= **先加入的副圖**（pane 索引最小者）
> - **被強制關閉副圖的 savedHeight**：下次重新勾選啟用時，一律以 `_defaultSubPaneHeight = 120px` 重新加入（不保留被關閉前的 savedHeight）

### 情境 D　拖拉底部 handle（容器整體縮放，等比縮放各 pane）

> **規則 2**：等比縮放，各 pane 依拖拉前佔比重新分配高度。

**等比縮放公式**

```
 ratio_main   = mainPaneHeight_old / totalContainerHeight_old
 ratio_sub[i] = subPaneHeight_old[i] / totalContainerHeight_old

 mainPaneHeight_new   = totalContainerHeight_new × ratio_main
 subPaneHeight_new[i] = totalContainerHeight_new × ratio_sub[i]
 （各結果 clamp 在最小值 60px 以上）
```


### 情境 J　固定容器下，副圖拉到最大

> 在容器高度固定的情況下，透過連續拖拉分隔線，將目標副圖高度最大化（其他 pane 各自壓至最小 60px）。

**前提**：total=420px，main=180，VOL=120，RSI=120

```
 最大化公式：VOL_max = total − _minMainPaneHeight − _minSubPaneHeight(RSI)
                     = 420 − 60 − 60 = 300px
```

**Step 1：往上拖主圖↔VOL 分隔線，主圖壓至最小值（60px）**

```
 【初始】  total=420  main=180  VOL=120  RSI=120

 ┌──────────────────────┐ ─┐                ┌──────────────────────┐ ─┐
 │  主圖   180px        │  │  往上拖 120px   │  主圖   60px         │ ─┘ ← 觸底
 │                      │ ─┘ ──────────→   ╠══════ 分隔線（已到頂）╣
 ╠══════ ↕ 主↔VOL 分隔線╣                  │ ⚙ ✕  VOL  sH=240   │
 │ ⚙ ✕  VOL  sH=120   │                  ├─────────────────────┤
 ├──────────────────────┤                  │  VOL 繪圖（放大）    │
 ╠══════ ↕ VOL↔RSI 分隔╣                  ╠══════ ↕ VOL↔RSI ════╣
 │ ⚙ ✕  RSI  sH=120   │                  │ ⚙ ✕  RSI  sH=120  │ ← 未變
 └──────────────────────┘                  └─────────────────────┘

 主圖已觸底，分隔線停止響應
```

**Step 2：往下拖 VOL↔RSI 分隔線，RSI 壓至最小值（60px）**

```
 【Step 1 後】  total=420  main=60  VOL=240  RSI=120

 ┌──────────────────────┐ ─┐                ┌──────────────────────┐ ─┐
 │  主圖   60px         │ ─┘  往下拖 60px   │  主圖   60px         │ ─┘ ← 不受影響
 ╠══════ 主↔VOL 分隔線 ══╣ ──────────→      ╠══════ 主↔VOL 分隔線 ══╣
 │ ⚙ ✕  VOL  240px    │                  │ ⚙ ✕  VOL  300px    │ ← 最大化！
 ├──────────────────────┤                  ├──────────────────────┤
 │  VOL 繪圖            │                  │  VOL 繪圖（最大化）   │
 ╠══════ ↕ VOL↔RSI ═════╣                  ╠══════ VOL↔RSI（已到底）╣
 │ ⚙ ✕  RSI  sH=120   │                  │ ⚙ ✕  RSI  60px     │ ← 觸底
 └──────────────────────┘                  └──────────────────────┘

 RSI 已觸底，分隔線停止響應
```

**最終狀態（VOL 最大化）**

```
 ┌──────────────────────────────────────────┐ total=420px（固定）
 │  主圖   60px（最小值，分隔線停止響應）     │
 ╠══════════════════════════════════════════╣
 │ ⚙ ✕  VOL   savedH = 300px              │ ← 最大化
 │                                          │
 │           VOL 繪圖（放大）               │
 │                                          │
 ╠══════════════════════════════════════════╣
 │ ⚙ ✕  RSI   savedH = 60px               │ ← 最小值（52px 工具列 + 8px 繪圖）
 └──────────────────────────────────────────┘
 ≡≡≡≡≡≡≡≡≡≡≡≡≡≡ 底部 handle ≡≡≡≡≡≡≡≡≡≡≡≡

 VOL_max = total − min(main) − min(RSI) = 420 − 60 − 60 = 300px ✓
```

> **注意**：`_minSubPaneHeight = 60px` 是副圖 pane 的最小高度（工具列與繪圖區整體包含於內）。  
> 副圖觸底後繪圖區幾乎不可見，但工具列仍正常顯示，使用者可從圖表管理或 ✕ 關閉。

## 五、邊界值行為摘要表

| 觸發條件 | 行為 | 說明 |
|----------|------|------|
| 新增副圖後 main **嚴格 < 60px** | 強制關閉最先開啟的副圖（cascade 直到 main ≥ 60px） | **不自動擴展容器** |
| 移除副圖 | main 增加 = 該副圖的 savedHeight | 容器不變 |
| 拖拉底部 handle 縮小 | 等比縮放，各 pane 結果 clamp ≥ 60px | 若縮放使 pane < 60px，以最小值保底 |
| 拖拉底部 handle 到最小值（420px）| 容器停止縮小；主圖填滿容器（不呼叫 setHeight，由 LW 原生填滿） | `_minTotalChartHeight = 420px` |
| 拖拉主↔副分隔線使 main = 60px | 分隔線停止響應（不能再往上拖） | VOL 已拿到最多空間 |
| 拖拉副↔副分隔線使某副圖 = 60px | 分隔線停止響應（不能再往同方向拖） | 副圖已拿到最多空間 |
| 全螢幕進入 | 記錄 `_preFullscreenHeight`，等比放大至螢幕 | |
| 全螢幕退出 | 還原 `totalContainerHeight = _preFullscreenHeight` | |
| 切換股票 | savedHeight 與 totalContainerHeight 完整保留 | 股票切換不重置高度 |
| 頁面重新整理 | totalContainerHeight = `_minTotalChartHeight = 420px` | |

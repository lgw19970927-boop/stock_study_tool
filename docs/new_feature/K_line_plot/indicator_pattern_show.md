### 📌 Part 1: New Features (新功能實作)

#### Feature A: K 線圖初始視角定位與完整歷史滑動 (Viewport & Panning)
這個功能涉及「資料集 (Dataset)」與「可視範圍 (Viewport/Visible Range)」的精準分離。
* **1. 初始視角定位**：畫面左側有「分析時間範圍」設定區塊（例如設定為 `2025/06/01 ~ 2025/12/31`）。假設今天為 `2026/03/16`，系統預設的 K 線顯示範圍比例為「1 年」。當使用者點擊篩選結果的某檔股票載入圖表時，**預設可視範圍的最右側（畫面上的最後一根 K 線）必須對齊「分析時間範圍的結束日期」**（即 2025/12/31）。因此，初始畫面呈現的 K 線 X 軸可視範圍應該要是 `2025/01/01 ~ 2025/12/31`，而非 `2025/03/15 ~ 2026/03/16`。
* **2. 保留完整的滑動能力 (不截斷資料)**：雖然初始視角定位在過去（2025 年底），但圖表載入的資料集必須是完整的（包含到今天的最新資料）。使用者依然可以自由地往左滑（查看 2025/01/01 之前的歷史）或往右滑（查看 2025/12/31 之後到今天的最新 K 線）。
* **技術重點**：只能透過類似 `setVisibleRange` 或 `dataZoom` 的 API 來改變「初始視圖範圍」，絕對不能在傳給圖表的資料源陣列中刪除或過濾掉全域的 K 線資料。

#### Feature B: 圖表左上角指標控制列與 Chart Setting 彈窗的雙向連動
為了確保圖表左上角控制列、圖表管理（Chart Setting）彈窗、以及實際畫布（Canvas）三者的狀態一致，必須採用**單一資料來源（Single Source of Truth）**的狀態管理模式。
* **操作 1（點選 ⚙️ 齒輪與特定設定頁面）**：點選左上角該指標的「齒輪」圖示，開啟 Chart Setting 彈窗，並自動切換到對應的指標設定頁籤。
  * **【重要UI實作細節】：在 Chart Setting 中點選或切換到 BOLL 時，必須切換到專屬的 BOLL 設定頁面。此頁面的 UI 設計請務必嚴格參考我提供的截圖（包含：計算週期、股票特性參數、MID/UPPER/LOWER 指標線的勾選框、線寬、顏色、不透明度設定等欄位）。**
* **操作 2（點選 ✖️ 叉叉）**：將該指標的全域狀態（isGlobalEnabled）設為 false。左上角該指標整列隱藏 ➔ 彈窗內主圖該指標的 Checkbox 取消勾選 ➔ 畫布上清除該指標所有線條。
* **操作 3（點選參數文字，如 MA 20）**：單獨反轉該參數線條的啟用狀態（isEnabled）。啟用時文字高亮並顯示數值，停用時文字變暗且不顯示數值 ➔ 彈窗內對應指標線 Checkbox 同步勾選/取消 ➔ 畫布動態繪製/移除該線條。
* **逆向連動（Modal to Top Bar）**：彈窗內的任何操作（勾選切換、數值/顏色修改），都必須即時反映至左上角控制列與圖表畫布上，不需等待重整。

#### Feature C: 型態辨識篩選結果的 K 線圖框選標註 (Pattern Annotation)
這是一個獨立的視覺標註圖層，用來高亮顯示符合「型態辨識」的特定 K 線區間。請參考我提供的圖片（含有 Switch 開關與白框的截圖）。
* **1. UI 開關控制**：畫面上方有一個「型態標註」的 Switch 開關。當開關為 ON 時，在圖表上繪製標註框（如截圖的白框）；當開關為 OFF 時，清除/隱藏該標註框。
* **2. 嚴格的時間範圍綁定**：這個標註框的座標（Start Date, End Date, Top Price, Bottom Price）必須**嚴格綁定於「分析時間範圍」內**（例如剛剛設定的 `2025/06/01 ~ 2025/12/31`）。
* **3. 視覺行為約束**：因為型態分析的範圍是綁定時間分析範圍的，當使用者在 Feature A 中往左或往右滑動，查看到 `2025/06/01 ~ 2025/12/31` 以外的 K 線時，不應渲染任何額外的型態標註框（亦即，標示框只會固定出現在當初設定的分析範圍內的特定 K 線上，跟著畫布一起平移，但不會因為滑到其他時間段就自動產生新的標註）。

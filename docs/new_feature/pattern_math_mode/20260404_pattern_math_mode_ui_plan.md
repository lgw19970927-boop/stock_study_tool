# 型態篩選 - 數學判斷模式 UI 設計方案

- **日期**：2026-04-04
- **現況**：型態篩選中，「盤整區」採用數學規則判斷（ATR 波動率 + 布林帶寬度），其餘型態（W底、頭肩頂、頭肩底、三角收斂）全部採用深度學習模型辨識。

---

## 一、背景說明

| 型態 | 現有方式 |
|------|----------|
| 盤整區 | 數學判斷 ✅ |
| W 底 | 模型辨識 |
| 頭肩頂 | 模型辨識 |
| 頭肩底 | 模型辨識 |
| 三角收斂 | 模型辨識 |

數學判斷優點：速度快、可解釋、不依賴模型權重。  
模型辨識優點：能捕捉非完美型態的模糊特徵。  
**設計目標**：讓使用者可按型態自由選擇「使用哪種模式」。

## 三、互動邏輯說明

```
使用者勾選型態 checkbox
        │
        ▼
型態卡片展開 → 顯示「識別模式」Radio（模型 / 數學）
        │
        ├─ 選「模型辨識」
        │       └─ 僅保留靈敏度 Slider（現有邏輯）
        │
        └─ 選「數學規則」
                └─ 顯示該型態對應的數學參數面板
                        （容忍誤差、回看週期等，依型態而異）
```

## 四、各型態數學判斷參數規劃

| 型態 | 建議數學規則 | 可調參數 |
|------|-------------|----------|
| W 底 | 兩個低點高度差 < 容忍率 × 平均低點，且中間反彈 > 最小幅度 | 容忍率 %、最小反彈幅度 %、回看週期 |
| 頭肩頂 | 三頂高度比較（左肩 ≈ 右肩 < 頭部），頸線斜率 | 肩部對稱容忍率 %、頸線最大斜率、回看週期 |
| 頭肩底 | 三底深度比較（左肩 ≈ 右肩 > 頭部） | 同上 |
| 三角收斂 | 高點線斜率 < 0、低點線斜率 > 0，兩線 R² > 閾值 | R² 閾值、收斂角度上限、回看週期 |
| 盤整區 | 現有：ATR 比率 + 布林帶寬度（已實作） | 靈敏度 Slider 對應現有參數 |

## 五、前端實作方案

### 5.1 HTML 結構調整（`pattern_panel.html`）

在每個 `.pattern-card` 的 `<span>型態名稱</span>` 後面加入隱藏的模式選擇區：

```html
<label class="pattern-card">
    <input type="checkbox" name="pattern" value="w_bottom">
    <div class="pattern-icon"><!-- SVG --></div>
    <span>W 底</span>
    <!-- 新增：勾選後展開的模式選擇 -->
    <div class="pattern-mode-panel is-hidden">
        <div class="pattern-mode-label">識別模式</div>
        <label class="pattern-mode-radio">
            <input type="radio" name="w_bottom_mode" value="model" checked> 模型辨識
        </label>
        <label class="pattern-mode-radio">
            <input type="radio" name="w_bottom_mode" value="math"> 數學規則
        </label>
        <!-- 數學模式參數面板（預設隱藏）-->
        <div class="pattern-math-params is-hidden">
            <div class="pattern-param-row">
                <label>容忍誤差</label>
                <input type="number" class="pattern-param-input" value="5" min="1" max="20">
                <span>%</span>
            </div>
            <div class="pattern-param-row">
                <label>回看週期</label>
                <input type="number" class="pattern-param-input" value="20" min="5" max="100">
            </div>
        </div>
    </div>
</label>
```

### 5.2 JS 邏輯（`pattern_manager.js`）

1. 監聽 `.pattern-card input[type="checkbox"]` 的 change 事件 → toggle `.pattern-mode-panel` 的 `is-hidden`
2. 監聽 `.pattern-mode-radio input[type="radio"]` → 若選「數學規則」則顯示 `.pattern-math-params`
3. 收集篩選條件時，對每個勾選型態判斷其 `mode`，輸出不同的後端 payload：
   - `mode: "model"` → 現有邏輯不變
   - `mode: "math"` → 附加數學參數 `{ tolerance_pct, lookback_n }`

### 5.3 後端 payload 格式

```json
{
  "patterns": [
    { "type": "w_bottom", "mode": "model" },
    { "type": "head_shoulders_top", "mode": "math", "params": { "shoulder_tolerance": 5, "lookback": 60 } }
  ]
}
```

## 六、風險與替代方案

| 風險 | 說明 | 緩解 |
|------|------|------|
| 卡片高度不一致 | 展開數學參數後高度不同，grid 版面可能錯位 | 改用 flex-column 的 `pattern-grid`，展開時各卡獨立撐高 |
| 數學規則未實作 | 後端目前只有盤整區有數學判斷 | 前端可先完成 UI，後端標記 `not_implemented` 時前端顯示「開發中」badge |
| 參數組合爆炸 | 每個型態有不同參數，維護成本高 | 統一為 `tolerance_pct` + `lookback` 兩個通用參數，足夠覆蓋多數型態 |

---

## 七、實作優先順序建議

1. **Phase A**（前端 UI skeleton）：HTML 結構 + CSS 展開動畫，不接 JS 邏輯  
2. **Phase B**（前端互動）：checkbox 展開 + radio 切換 + 參數收集  
3. **Phase C**（後端）：補充 W底、頭肩頂/底、三角收斂 的數學判斷演算法  
4. **Phase D**（整合）：前後端 payload 串接 + 測試

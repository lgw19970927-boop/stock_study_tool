# 模擬交易回測頁面架構設計 (詳細完整版)

頁面整體分為 **左側（回測參數設定面板）** 與 **右側（回測報告與分析面板）** 兩大區塊。左側面板負責定義後端模擬迴圈的所有規則，右側面板則負責接收後端運算結果並進行視覺化渲染，風格排版類似股票篩選頁面的左右側架構。

## ▍左側面板：回測參數設定 (Settings Panel)

此區塊為模擬交易規則的控制中心，所有條件皆在後端進行模擬，獨立運作。

### 1. 基本回測參數 (Basic Parameters)
* **回測期間 (Backtest Period)：**
  * 快捷按鈕列：`1D` / `1W` / `1M` / `3M` / `6M` / `1Y` / `自訂`（風格與股票篩選頁面的「分析時間範圍」一致）。
  * 選擇「自訂」時，展開開始日期 (`start_date`) 與結束日期 (`end_date`) 選擇器。
* **執行頻率 (Execution Frequency)：** * 下拉選單：1分 / 3分 / 5分 / 15分 / 30分 / 1H / 4H / 日 (1D) / 週 (1W) / 月 (1M) / 季。
  * *說明：決定後端回測引擎每次推進的時間單位。*
* **交易方向 (Trade Direction)：** * 單選按鈕：作多 (Long) / 作空 (Short)。
* **執行商品範圍 (Execution Instrument Range)：**
  * 選項卡或下拉選單，決定回測的母體池（僅限美股）：
    1. **股票篩選策略：** 匯入「股票篩選」分頁中已儲存的「我的策略」條件，作為動態股票池（每個時間節點只交易符合該策略的股票）。
    2. **自訂股票清單：** 手動輸入或匯入特定股票代碼（例如：AAPL, NVDA, MSFT）。
    3. **全市場：** 針對美股全市場進行掃描。

### 2. 進場設定 (Entry Settings)
* **進場觸發條件 (Entry Triggers)：**
  * UI 支援**摘要模式 / 編輯模式**雙模式切換（與現有股票篩選器的指標/型態設定一致）：
    * **摘要模式 (Summary Mode)**：顯示簡明條件字串（例如：`MA-日K: MA20 > MA60`），右側附帶「✏ 編輯」與「🗑 刪除」Icon。
    * **編輯模式 (Edit Mode)**：點擊「✏ 編輯」或「+ 新增進場條件」時展開卡片，提供指標 / 型態選擇、週期、比較運算子及數值等下拉選單。
* **進場限制 (Entry Restrictions)：**
  * **開盤跳空設定**：組合式選項 — `開盤跳空` `[向上 / 向下 ▼]` `達` `[輸入框]` `%` 即 `[交易 / 不交易 ▼]`
  * **特定時間區段**：組合式選項 — `開盤後` `[輸入框]` `[分鐘 / 小時 ▼]` `[交易 / 不交易 ▼]`
  * *說明：兩組限制條件為「AND」關係，皆啟用時須同時滿足。*
* **進場價格機制 (Entry Price execution)：**
  * 單選按鈕：`次期開盤價 (Next Open)` / `當期收盤價 (Current Close)` / `條件觸發價 (Trigger Price)`
  * *「條件觸發價」說明：適用於僅有日線資料的環境下模擬「盤中突破」。若當日高 / 低點穿越觸發條件的目標價位，則直接以該突破價格作為進場成本（無需等待下根 K 線開盤）。*
* **最大同時持有檔數 (Max Open Positions)：** * 數字輸入框（留空或輸入 0 代表無限制）。

### 3. 出場設定 (Exit Settings)
* **條件出場 (Condition-based Exit)：**
  * UI 支援**摘要模式 / 編輯模式**雙模式切換（與進場觸發條件一致）。
  * 允許設定技術指標或型態作為出場觸發（例如：跌破 20MA、RSI 死亡交叉）。
* **固定停利 / 停損 (Stop Profit / Stop Loss)：**
  * 停利 (Take Profit)：核取方塊 `[ ]` 啟用，輸入 `%` 數。
  * 停損 (Stop Loss)：核取方塊 `[ ]` 啟用，輸入 `%` 數。
* **時間停損 (Time Stop)：**
  * 最大持有時間：核取方塊 `[ ]` 啟用，輸入 `N` 期（依據上方設定的「執行頻率」單位計算）。
* **出場價格機制 (Exit Price execution)：**
  * 單選按鈕：`次期開盤價 (Next Open)` / `當期收盤價 (Current Close)` / `條件觸發價 (Trigger Price)`
  * *「條件觸發價」說明：同進場機制，適用日線資料下模擬當日觸及出場條件價位時直接出場（例如跌破特定支撐線即以穿破價出場）。*

### 4. 資金與交易設定 (Capital & Trading Settings)
* **初始資金 (Initial Capital)：** * 數字輸入框（例如：$1,000,000）。
* **資金與風險管理模組 (Risk Management Module) [Optional]：**
  * 核取方塊 `[ ]` 啟用 / 停用。
  * *說明：啟用後，後端將自動套用與前端「資金與風險管理」分頁完全相同的數學邏輯。系統會依據設定的「單筆停損容忍度 (%)」自動計算每次進場的建議股數，並監控帳戶總風險。若單筆進場會導致帳戶總風險超標，則自動濾除該筆交易。*
* **交易費用 (Trading Costs)：** * 數字輸入框與單位下拉選單：輸入 `%` 數或單筆固定金額（貼合美股券商常見的計費特性）。

## ▍核心概念說明：篩選頻率 vs 執行頻率 vs 橋接邏輯

下表說明三個相關但獨立的概念，避免使用者在匯入篩選策略時產生混淆：

| 層級 | 名稱 | 定義位置 | 說明 |
|------|------|----------|------|
| 1 | **篩選頻率 (Screening Frequency)** | 股票篩選頁面（儲存在「我的策略」中） | 定義「這個選股策略多久執行一次市場掃描」（例如：每日、每週）。這是策略本身的一部分，與條件內使用的 K 棒週期無關（例如：可以每週掃描，但條件看日 K 線）。 |
| 2 | **執行頻率 (Execution Frequency)** | 模擬回測頁面 | 定義「回測引擎每次前進的時間單位」，決定系統多久去檢查一次進出場條件是否成立。這是回測的屬性，與選股策略無關。 |
| 3 | **橋接邏輯 (Bridge Logic)** | 匯入策略時的處理 | 當回測頁面匯入某個篩選策略時，該策略的「篩選頻率」預填為目前設定的執行頻率，但使用者可以 override。股票篩選頁面本身是監控當前時間點的工具，策略應為自洽的完整描述，因此篩選頻率應保留在策略中，不應從篩選頁面移除。 |

## ▍右側面板：回測報告與分析 (Backtest Report Panel)

點擊「開始回測」後，右側面板將渲染完整報告。建議採用 **頁籤 (Tabs)** 切換或 **區塊向下滾動** 的方式排版。

### 1. 回測基本資訊表頭 (Report Header)
設計為報告最上方的資訊卡，便於快速確認條件或截圖：
* **回測資料範圍：** `YYYY/MM/DD` - `YYYY/MM/DD` (含執行頻率)。
* **回測執行時間：** 報告產生的時間戳記與後端運算耗時。
* **回測執行商品：** 顯示範圍名稱（例：美股全部 / 自訂清單：科技七巨頭），並註記回測成功與失敗的檔數。
* **交易設定摘要：** 顯示交易方向、交易費用、最大持有檔數限制、進/出場價格基準。

### 2. 整體統計表 (Overall Statistics)
核心績效數據總覽，建議包含正負值顏色標示（綠漲紅跌）：
* **交易頻次：** 總交易次數 | 獲利次數 | 虧損次數
* **核心指標：** 勝率 (Win Rate %) | 盈虧比 (Profit/Loss Ratio)
* **報酬與損益分析：** * 總報酬率 (Total Return %)
  * 複合年化成長率 (CAGR)
  * 平均報酬率-單筆 (Avg. Return per Trade %)
  * 平均獲利% (Avg. Win %) | 平均損失% (Avg. Loss %)
  * 平均獲利金額 (Avg. Win $) | 平均損失金額 (Avg. Loss $)
  * 最大獲利-單筆 (Max Profit per Trade %) | 最大虧損-單筆 (Max Loss per Trade %)
* **風險與極端值：** * 最大回撤 (Max Drawdown)
  * 最大連續獲利次數 | 最大連續虧損次數
  * 最大區間獲利率 (Max Interval Profit %) | 最大區間虧損率 (Max Interval Loss %)

### 3. 淨值曲線圖 (Equity Curve)
* **圖表類型：** 互動式折線圖或面積圖。
* **X 軸：** 時間軸（依照回測期間）。
* **Y 軸：** 總報酬率 (%) 或 帳戶總淨值金額。
* **互動功能：** 滑鼠懸停可顯示特定時間點的淨值，圖表中應包含 0% 水平基準線。

### 4. 商品統計表 (Symbol Statistics)
以「單檔股票」為維度的數據表格 (Data Grid)，可點擊欄位標題進行排序：
* 欄位：`股票代碼` | `交易次數` | `勝率` | `總貢獻損益金額` | `總貢獻損益 %`

### 5. 交易明細清單 (Trade Details List)
詳細紀錄每一筆交易的資料表，支援篩選與排序：
* 欄位：
  * `股票代碼`
  * `進場時間` | `進場價格`
  * `出場時間` | `出場價格`
  * `進場原因` (例如：MACD 黃金交叉)
  * `出場原因` (例如：觸發 8% 停損 / 跌破月線)
  * `持有期數`
  * `單筆損益 %` | `單筆損益金額`

### 6. 交易明細圖表 (Interactive Trade Chart)
**[重要互動]** 點擊上方「交易明細清單」中的任一筆列 (Row) 時，此圖表會自動更新為該筆交易的視覺化覆盤：
* **主圖 (K線圖)：** 顯示該檔股票的 K 線走勢。明確標記「進場點 (Buy)」與「出場點 (Sell)」的圖示，並畫出連接進出點的交易線段。
* **副圖 (指標與量能)：** 顯示相關的技術指標線圖與成交量柱狀圖，並畫出對齊進出場時間點的垂直參考線。

## ▍設計審查：潛在邏輯衝突與模糊點

### 🔴 衝突 1：執行頻率 vs 股票篩選策略的動態池更新成本
**問題點：** 「執行商品範圍」選擇「股票篩選策略」時，說明指出「每個時間節點只交易符合該策略的股票」。若執行頻率為 `1min` 或 `5min`，回測期間數年，每個時間步都需要重新掃描整個指標篩選策略，計算量會是 `O(T × N × I)` 完全非線性。

**建議：** 需要明確定義股票池的**更新頻率**，例如：
- 每個「執行頻率」步驟更新一次（高成本）
- 每天更新一次（固定）
- 只在回測開始時建立一次靜態池（最快最簡單）

建議在 UI 增加一個選項：「動態更新（每步）」/ 「每日更新」/ 「靜態池（回測開始時建立一次）」。

---

### 🔴 衝突 2：最大持倉數 (Max Open Positions) 與風險管理模組的優先順序未定義
**問題點：** 若「最大同時持有檔數」設為 10，且「資金與風險管理模組」也啟用，兩個系統都可能各自決定「要不要進場」，但優先順序不明：是「先達到 10 檔就停」還是「先計算風險預算再決定是否達到 10 檔限制」？

**建議：** 明確定義執行順序：
1. 觸發進場條件
2. 飛過風險管理模組計算倉位
3. 若通過風險預算，才檢查是否超過最大持倉數
4. 全部通過才實際進場

---

### 🟡 模糊 1：同一個 bar 的進場與出場觸發衝突
**問題點：** 若進場設定為「當期收盤價」，出場設定也為「當期收盤價」，且兩個條件在同一根 K 棒上同時觸發，會產生「同一根棒開倉又平倉，損益 ≈ 0（扣手續費）」的無效交易。

**建議：** 規定若進場條件與出場條件在同一根 K 棒被同時觸發，**優先執行進場**，出場條件延後至下一個時間步驟重新判斷。

---

### 🟡 模糊 2：時間停損的顯示單位不夠直觀
**問題點：** 目前說明為「輸入 N 期（依照執行頻率計算）」。若頻率為「週 (1W)」而 N=12，使用者需要在心中換算為 12 週 = 約 3 個月。

**建議：** 輸入框旁邊即時顯示換算結果，例如：
> 最大持有時間：`[ 12 ]` 期　≈ **12 週（約 3 個月）**

---

### 🟡 模糊 3：回測期間資料可用性驗證缺失
**問題點：** 若使用者設定 `start_date = 2010-01-01`，但資料庫的美股歷史資料最早只有 2019 年，系統將無聲地跑出錯誤或空白報告，沒有明確說明應如何處理。

**建議：** 在右側面板頂部（或 Start 按鈕旁）提示可用資料時間範圍。若設定的 start_date 超出資料範圍，在按下「開始回測」時顯示警告，並自動調整或阻止執行。

---

### 🟡 模糊 4：CAGR 的計算邊界情況
**問題點：** CAGR 定義為 `(期末值/期初值)^(1/年數) - 1`，若回測期間 < 1 年（例如只有 3 個月），算出的 CAGR 年化率可能非常誇張（如 +1200%），容易誤導使用者。

**建議：** 若回測期間 < 12 個月，將「複合年化成長率 (CAGR)」標注一個 `⚠ 回測期間不足 1 年，僅供參考` 的提示，或改為顯示「總回報率」主要指標。

---

### 🟡 模糊 5：商品統計表的零交易股票顯示策略
**問題點：** 若某隻股票在回測期間從未觸發進場條件（0 次交易），它是否應出現在「商品統計表」中？若不顯示，使用者無法得知哪些股票「在池中但沒有成交」。

**建議：** 提供一個切換選項「只顯示有交易紀錄的股票 / 顯示全部股票池清單」，預設只顯示有成交的股票。

---

### 版面設計要點
1. **左側面板**：與篩選頁面左側一致，同樣使用 `position: sticky; top: 0`，高度 `100vh - topbar`，overflow-y: auto 捲動；各設定區塊使用折疊 `<details>` 或分隔線區塊。
2. **回測資訊表頭**：4 個數字卡片橫向排列（同現有篩選結果上方的 `統計結果` row 風格）。
3. **整體統計**：使用 8～12 個小卡片 (`grid: repeat(auto-fill, minmax(120px, 1fr))`)，正值綠色 / 負值紅色，沿用現有色彩變數 `--color-up` / `--color-down`。
4. **淨值曲線圖**：採用 LightweightCharts `addAreaSeries`（與 K 線圖保持統一技術棧），高度約 200–280px，固定顯示 0% 水平基準線。
5. **商品統計表**：與現有篩選結果的股票列表一致，支援欄位點擊排序；行點擊後自動切換至「交易明細 Tab」並預篩選該股票的明細。
6. **交易明細 Tab 上下分割**：上方明細列表（~40% 高度）+ 下方 K 線圖（~60%），中間加入**垂直拖曳調整高度**的 resize handle（與現有篩選頁面一致）。
7. **「開始回測」按鈕**：固定在左側面板底部，透過 sticky 定位確保隨時可見，使用 `--accent` 顏色；執行中顯示進度條（SSE streaming），支援中途停止。

# 模擬交易回測 - 進階直覺化 UI/UX 佈局提案(參考看看即可)

## 1. 傳統「左右側固定面板」的痛點分析
雖然我們最初評估了傳統的「左側設定、右側圖表」架構，但套用到**「模擬交易回測」**時會產生幾個不可避免的痛點：
1. **表單疲勞**：把所有（基本、進場、出場、資金、風控）選項擠在狭長的左側面板中，會大幅增加垂直滾動的操作成本。
2. **空間排擠效應**：將報表限縮在右側 70% 的畫布中，難以完美展開「淨值曲線」、「交易明細」等高密度數據，損害覆盤體驗。

---

## 2. 方案一：語意化看板工作流 (No-Code SaaS Style)
**參考對象：現代化 Fintech SaaS、量化積木無程式碼工具。**
徹底捨棄左右分割佈局，將設定介面變成「滿版水平卡片直列」，執行後再切換至純報表。

### 視圖一：策略建構中心 (滿版展示)
將生硬的下拉選單包裝成「符合自然語言閱讀習慣」的短句。

---

**【設計細節與優勢】**
1. **空間解放**：每個獨立區塊都有廣大的空間，可以輕鬆容納進階選項（如手續費、觸發價等），而不會像左側面板那樣擠成一團。
2. **語意化 UI**：以「當...時，就...」的句型包裹下單邏輯，填寫表單就像在寫交易日記，極度降低新手門檻。
3. **模組化擴充**：未來若有新的「濾網限制」或「複雜進場條件」，只需在卡片內按下 `(➕ 新增)` 即可垂直擴展積木。

### 視圖二：【回測實驗室與報表 (Analyze Mode)】- 螢幕滿版
一旦按下「執行」，系統進入「運算進度條」狀態（同樣採用毛玻璃遮罩），完成後自動切換至此視圖。

├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  ┌─ 核心指標總覽 (綜合績效) ──────────────────────────────────────────┐ │
│  │                                                                    │ │
│  │   勝率 55%  |  總報酬 +125%  |  最大回撤 -14%  |  交易次數 142次       │ │
│  └────────────────────────────────────────────────────────────────────┘ │
│                                                                        │
│  ┌─ 淨值曲線圖 (可全螢幕互動) ────────────────────────────────────────┐ │

│  └────────────────────────────────────────────────────────────────────┘ │

**【設計細節與優勢】**
1. **零干擾分析**：使用者擁有 100% 的水平寬度來查看複雜的數據表（商品統計、交易明細）與線圖，大幅減少因螢幕寬度不足而需要橫向滾動 (horizontal scroll) 的痛苦。
2. **參數快照帶 (Snapshot Ribbon)**：雖然設定介面隱藏了，但頂部永遠有一列「標籤化」的參數摘要，確保分析時不會忘記當前的測試條件。
3. **無縫修正循環**：發現績效不好，點擊左上角的 `⬅ [返回修改策略]`，立刻切換回「滿版看板模式」進行調整，形成流暢的 (設定 ➔ 執行 ➔ 分析 ➔ 修改) 工作流。

## 3. 方案二：專業多視窗與高密度主控台 (台系 XQ 佈局)
**參考對象：以 XQ 全球贏家的「介面架構」重新包裝您的核心需求。**
這套方案 **完全遵守《模擬交易回測.md》指定的每一項功能**，但套用 XQ「極大化資訊密度與彈出式報告」的介面哲學。捨棄了語意化，改用**高密度的樹狀目錄 + 設定儀表板**，搭配獨立彈出的多頁籤大報表。

### 視圖一：策略主控台 (固定於主畫面)
將所有您指定的參數高密度地收納於此，操作動線俐落專業。

## 4. 方案三：主圖連動型底部抽屜 (美系 TradingView / IDE Style)
**參考對象：TradingView Strategy Tester、QuantConnect 介面。**
歐美最核心的軟體設計哲學是：**「K 線圖與策略的買賣點標記絕對不可分割」**。
在這種佈局中，畫面**上方永遠是極大化的主視覺 K 線圖**。所有的回測設定會隱藏在頂端的小型「齒輪彈出框 (Gear Overlay Dialog)」中；而最關鍵的報表與交易明細，則是做成一個可以上下托曳伸縮的 **「底部抽屜 (Bottom Drawer Panel)」**。

### 視圖總覽：上方主圖為王、下方抽屜為輔
```text
┌────────────────────────────────────────────────────────────────────────┐
│ [ 突破策略 ▼ ] [ ⚙ 設定 ] [ 👁 顯示/隱藏買賣標記 ]            [ ⛶ 全螢幕 ] │
├────────────────────────────────────────────────────────────────────────┤
│ 📈 【 全螢幕互動 K 線圖 (佔據畫面頂部 65% 空間) 】                             │
│                                        [ ⬇ 賣出 ]                      │
│      ■ ▇ ▆                               |                             │
│       |  ▆ ▇ ■                           ■                             │
│        [ ⬆ 買入 ]                                                      │
│                                                                        │
│  (當您點擊下方抽屜明細的任何一筆交易，上方的 K 線圖會「自動平移定位」到該筆交易日)    │
│                                                                        │
├────────────────────────────────────────────────────────────────────────┤
│ ▅▅▅▅▅▅▅▅▅▅▅▅▅▅ ⬆ [點拉拖曳以改變底部抽屜高度] ⬇ ▅▅▅▅▅▅▅▅▅▅▅▅▅▅▅▅▅▅▅ │
├────────────────────────────────────────────────────────────────────────┤
│ 📊 策略測試器 (Strategy Tester)                [ ⏹ 停止執行並保留結果 ] │
│ 頁籤：[ 績效總覽 (Overview) ] [ 交易列表 (List of Trades) ]              │
├────────────────────────────────────────────────────────────────────────┤
│ 淨利         總交易次數      勝率        最大區間虧損      獲利因子             │
│ $ 125,000    142           55.3%      14.2%         2.15               │
├────────────────────────────────────────────────────────────────────────┤
│ 淨值曲線 (微型預覽):   / \_____/ \_______                                │
└────────────────────────────────────────────────────────────────────────┘
```

**【設定參數的方式：覆蓋式設定窗 (Gear Overlay Dialog) 】**
點選頂端 `[ ⚙ 設定 ]` 按鈕，就會立刻跳出一個迷你面板來修改《模擬交易回測.md》的數值。

```text
  ┌──────────────────────────────────────────────┐
  │ ⚙ 策略設定與環境參數                        ✖ │
  ├──────────────────────────────────────────────┤
  │ 頁籤：[ 環境與進出場條件 ]  [ 資金與風控設定 ]    │
  │ ──────────────────────────────────────────── │
  │ 初始資金：[ $ 1,000,000 ]                   │
  │ 每筆資金：[ 依比例 10% ▼] 小於等同 100,000    │
  │ 最大持倉：[ 10 ] 檔                         │
  │ 停損(%): [ 8 ]        停利(%): [ 15 ]      │
  │ 時間停損: [ 10 ] 期                         │
  │                                              │
  │ ──────────────────────────────────────────── │
  │                        [ 套用並重新計算 ↺ ]  │
  └──────────────────────────────────────────────┘
```

**【方案三的核心優勢】**
1. **所見即所得的高速迭代 (WYSIWYG)**：這是目前歐美最高級的量化 IDE 體驗。使用者打開 `[ ⚙ 設定 ]` 修改「停損點 8% 變 10%」後按套用，底部的抽屜績效跟上方 K 線圖的進出場箭頭會**幾乎瞬間連動更新**。不需要獨立視窗跳來跳去，打造「邊調參數、邊看圖形」的高速印證循環體驗。
2. **圖像與數據的強連結**：傳統報表很難知道「這筆虧大錢的交易發生時長什麼樣子」，但在 TradingView 架構中，只要在清單點擊該筆交易，主圖立刻帶你傳送到那根 K 線，用肉眼做立體分析。
3. **對單螢幕極度友善**：不需要開兩個視窗、也不需要分割左右擠壓空間。最精華的圖表享受全版寬度，報表則以「抽屜」形式平時收合、需要再拉長拉高，能最大化單一螢幕幕效益。

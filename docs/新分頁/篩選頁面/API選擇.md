# 📈 系統回測資料庫建置策略 (全市場 + 長歷史 精確修正版)

**架構核心理念：** 確保資料庫 100% 涵蓋全市場（約 18,000 檔，含主板與 OTC 粉單市場）。日線級別利用免費工具的極限深度打底；分線/時線級別則針對「至少 10 年歷史」的偏好，提供**最低成本的單月買斷戰術**，徹底解決免費 API 歷史過短的致命傷。

---

## 📅 級別一：`1d` (日線)
**🎯 系統目標：** 涵蓋全市場 (~18,000 檔) ＋ 超長歷史 ＋ 完全免費。

| 獲取階段 | 推薦來源 / API | 費用 | 歷史深度預期 | 執行策略說明 |
| :--- | :--- | :--- | :--- | :--- |
| **歷史打底 (一次性)** | **yfinance**<br>(`period="max"`) | **免費** | 依股票上市日<br>(可達 30 年以上) | **取代 Stooq 的全自動化方案**。雖然 `yfinance` 有速率限制，但它是免費資源中 OTC 涵蓋最廣的。建議寫一支 Python 背景腳本，設定安全的延遲（如每批次休眠 5 秒），花一個週末的時間將 18,000 檔股票的終身日K一次性抓進 MySQL。 |
| **每日更新 (排程)** | **yfinance** 或<br>**MarketData.app** | **免費** | 最新 1 天 | 歷史打底完成後，每日收盤透過既有的排程進行增量更新。若 `yfinance` 頻繁阻擋，可註冊 `MarketData.app` (每日 10,000 次免費請求額度) 作為備援切換。 |

## ⏱️ 級別二：`1h` (時線)、`5m` (5分線)、`1m` (1分線)
**🎯 系統目標：** 涵蓋全市場 (~18,000 檔) ＋ 至少 5 年歷史 ＋ 花費最低。
**💡 現實狀況：** 免費 API 的分K極限為 7 天 ~ 60 天。要滿足「5 年歷史 + 全市場」，必須採用**低成本單月買斷**的策略。

### 1. 歷史打底 (一次性)
**強烈建議：** 選擇符合預算的付費方案訂閱 **1 個月**，寫腳本全速將 18,000 檔股票的歷史分 K 拉回本地資料庫，抓完後立刻取消訂閱。

| 方案類型 | 推薦 API | 費用 (單月) | 歷史深度預期 | 執行策略說明 |
| :--- | :--- | :--- | :--- | :--- |
| **5年低標達陣** | **FinancialModelingPrep**<br>(Starter 方案) | **$19** | **精準 5 年** | **最符合你偏好的高 CP 值選擇**。只要不到 600 台幣，就能解鎖全市場包含 OTC 的 1m/5m/1h 數據，且歷史剛好給滿 5 年。利用這一個月把 5 年資料全數抓回本地。 |
| **長歷史解鎖** | **FinancialModelingPrep**<br>(Premium 方案) | **$49** | **長達 30 年** | 如果你後來發現 5 年不夠，想回測更早的週期，升級到 $49 方案可以解鎖長達 30 年的日內數據。 |
| *(免費妥協)* | *yfinance* | *免費* | *1m: 7天<br>5m: 60天<br>1h: 730天* | *如果完全不花錢，這就是地球上免費 OTC 分K的極限。只能把現有極限抓下來，以後靠自己每天更新慢慢存。* |

*(註：已排除 Polygon.io，因其包含長歷史的 Advanced 方案高達 $199/月，嚴重偏離「盡量花費最少」的核心原則。)*

### 2. 每日更新 (排程)
打底完成（並退訂付費 API）後，依靠伺服器的每日排程，自動把全市場最新的分K/時K補進資料庫，形成永續更新的自有資料庫。

| 方案類型 | 推薦 API | 費用 | 涵蓋範圍 | 執行策略說明 |
| :--- | :--- | :--- | :--- | :--- |
| **開源免費流** | **yfinance** +<br>**Webull API** (非官方) | **免費** | 上市 + OTC | **系統長治久安的關鍵**。利用微牛 (Webull) 非官方套件強大的 OTC 行情報價，與 yfinance 進行雙備援輪替。<br><br>**開發關鍵**：18,000 檔股票每日更新量太大，必須實作「活躍度分級」。日均量大的前 5000 檔每日更新，冷門無交易的 OTC 水餃股改為每週更新一次，以避開免費 API 的封鎖機制。 |

## 1. 設計背景與核心痛點

現有表格採用固定欄位（停損價位1、停損價位2、出場價位1、出場價位2），問題在於：
- **彈性不足**：N 批次停損/出場需要動態欄位，固定 2 欄無法支援分批停利規劃
- **無狀態管理**：無法區分「尚在規劃」vs「已進場持倉」vs「已全部出場」
- **無方向性**：只支援做多，無做空計算
- **欄位橫向過寬**：停損 2 欄 + 出場 4 欄造成表格需橫向捲動

本次改版採用「**同一列內垂直擴充 (Vertical Expansion inside Cell)**」設計，維持橫向 7 大欄的精簡結構，內部批次行在儲存格中往下堆疊。

## 4. 欄位定義：Input / Output 與計算公式

### 4.1 【狀態】欄

| 子欄位 | 類型 | 說明 |
|:---|:---|:---|
| 狀態徽章 | Output | 自動判定（見第 5 節狀態機） |
| 已進場 (Executed) | Input | Checkbox / Toggle。觸發狀態機切換 |

---

### 4.2 【持股】欄

| 子欄位 | 類型 | 說明 |
|:---|:---|:---|
| 股票代碼 (Ticker) | Input | 文字輸入，如 `NVDA`、`AAPL` |
| 方向 (Direction) | Input | 「多 ↑ / 空 ↓」Toggle 按鈕。預設為「多」|

---

### 4.3 【進場規劃】欄

| 子欄位 | 類型 | 公式 / 說明 |
|:---|:---|:---|
| 買入均價 (Avg Price) | Input | 每股買入價格，單位美元 |
| 預計倉位比例 (Capital %) | Input | 佔初始資金的比例，如 10 代表 10% |
| **建議股數** (Suggested Shares) | **Output** | 見公式 F-1（無停損輸入時顯示 `--`）|
| **控風倉位%** (Risk-Controlled Capital %) | **Output** | 見公式 F-2 |

**公式 F-1：建議股數**

$$
\text{建議股數} = \text{INT}\!\left(\frac{\text{Capital\%} \times \text{Capital} \times \max\!\left(1,\ \dfrac{\text{MaxRisk\%}}{|\text{平均停損\%}|}\right)}{\text{買入均價}}\right)
$$

- `INT()`：無條件捨去（只能買整股）
- 當 `|平均停損%| >= MaxRisk%` 時乘數 = 1（不放大倉位）
- 當 `|平均停損%| < MaxRisk%` 時乘數 = `MaxRisk% / |平均停損%|`（放大至風險上限）
- **邊界條件**：若平均停損% = 0（尚未輸入停損），顯示 `--`

**公式 F-2：控風倉位%**

$$
\text{控風倉位\%} = \frac{\text{建議股數} \times \text{買入均價}}{\text{Capital}} \times 100
$$

---

### 4.4 【停損設定】欄（支援動態 N 批次）

每一批次包含：

| 子欄位 | 類型 | 說明 |
|:---|:---|:---|
| 停損價位 (Stop Loss Price) | Input | 每股停損觸發價格 |
| 停損倉位佔比% (Stop Loss Batch %) | Input / Output | **最後一批** 為 Output（自動鎖定）；其餘為 Input |
| **平均停損%** (Avg Stop Loss %) | **Output** | 見公式 F-3，壓在儲存格底部 |

**公式 F-3：平均停損%**

$$
\text{平均停損\%} = \sum_{i=1}^{N}\!\left[\text{dir} \times \frac{\text{停損價}_i - \text{買入均價}}{\text{買入均價}} \times \text{批次佔比}_i\right] \times 100
$$

- `dir`：多 = +1，空 = -1（使得做多停損結果為負值，做空停損結果為正值）
- 最後一批佔比 = $100\% - \sum_{i=1}^{N-1}\!\text{批次佔比}_i$（自動計算，readonly）

**批次管理規則：**
- 初始：1 筆，佔比 = 100%，鎖定
- 新增：舊最後一批批次解鎖，新批次插入，新最後一批鎖定（自動計算剩餘%）
- 刪除非最後批：重新計算最後一批鎖定值
- 最少保留 1 批次（不可刪到為空）

---

### 4.5 【實際出場】欄（支援動態 N 批次）

每一批次包含：

| 子欄位 | 類型 | 說明 |
|:---|:---|:---|
| 平倉出場價位 (Exit Price) | Input | 每股出場價格 |
| 平倉出場倉位佔比% (Exit Batch %) | Input / Output | **最後一批** 為 Output（自動鎖定）；其餘為 Input |
| **平均損益%** (Avg P/L %) | **Output** | 見公式 F-4，壓在儲存格底部 |

**公式 F-4：平均損益%**

$$
\text{平均損益\%} = \sum_{i=1}^{M}\!\left[\text{dir} \times \frac{\text{出場價}_i - \text{買入均價}}{\text{買入均價}} \times \text{批次佔比}_i\right] \times 100
$$

- `dir`：多 = +1，空 = -1
- 最後一批佔比 = $100\% - \sum_{i=1}^{M-1}\!\text{批次佔比}_i$（自動計算，readonly）
- **邊界條件**：若無任何出場價位輸入，顯示 `--`

**狀態連動：**
- 🟡「規劃」狀態：整欄灰化（opacity 0.4），顯示「— 等待出場 —」文字；新增按鈕也灰化、不可點擊
- 🟢「持倉」/ ⚪「已結案」：解鎖，可正常操作
- ⚪「已結案」（出場佔比累計達 100%）：【停損設定】欄整體 readonly，灰化鎖定

---

### 4.6 【交易績效】欄

| 子欄位 | 類型 | 公式 / 說明 |
|:---|:---|:---|
| **盈虧比 (R/R Ratio)** | **Output** | 見公式 F-5 |
| **實際損益金額 (P/L Amount)** | **Output** | 見公式 F-6 |

**公式 F-5：盈虧比**

$$
\text{R/R Ratio} = \frac{\text{平均損益\%}}{|\text{平均停損\%}|}
$$

- 計算前提：平均損益% ≠ `--` 且平均停損% ≠ 0
- 否則顯示 `--`
- 色彩：> 1 顯示綠色（`#00d4aa`）；< 1 顯示紅色（`#f87171`）；= 1 顯示中性色

**公式 F-6：實際損益金額**

$$
\text{損益金額} = \text{平均損益\%} \times \text{建議股數} \times \text{買入均價}
$$

- 前提：平均損益% ≠ `--` 且建議股數 ≠ `--`
- 正值前綴 `+`，負值前綴 `-`；格式如 `+$15,632`
- 色彩：正值綠色，負值紅色

---

### 4.7 【帳戶貢獻】欄

| 子欄位 | 類型 | 公式 / 說明 |
|:---|:---|:---|
| **帳戶貢獻%** | **Output** | 見公式 F-7 |
| 顯示標籤 | Output | 依狀態切換（見下方） |

**公式 F-7：帳戶貢獻%**

$$
\text{帳戶貢獻\%} =
\begin{cases}
\text{平均停損\%} \times \text{控風倉位\%} & \text{if 無出場資料（規劃中）} \\
\text{平均損益\%} \times \text{控風倉位\%} & \text{if 有出場資料（持倉/已結案）}
\end{cases}
$$

**狀態標籤規則：**

| 條件 | 顯示標籤 | 色彩 |
|:---|:---|:---|
| 規劃 + 帳戶貢獻% < 0 | ⚠️ 預估風險 | 橘/紅（`#f87171`）|
| 持倉 + 帳戶貢獻% > 0 | 📈 實際獲利 | 綠色（`#00d4aa`）|
| 持倉 + 帳戶貢獻% < 0 | 📉 實際虧損 | 紅色（`#f87171`）|
| 已結案 + 正值 | 📈 已實現獲利 | 綠色 |
| 已結案 + 負值 | 📉 已實現虧損 | 紅色 |

## 6. 多空方向（Direction）計算邏輯

方向乘數 `dir`：
- 做多（多 ↑）：`dir = +1`
- 做空（空 ↓）：`dir = -1`

**做多範例（dir = +1）：**
- 買入均價 $850，停損 $782 → 停損% = +1 × ($782 - $850) / $850 = **-8.0%** ✓（負值正確）
- 買入均價 $850，出場 $950 → 損益% = +1 × ($950 - $850) / $850 = **+11.76%** ✓（正值正確）

**做空範例（dir = -1）：**
- 融券均價 $850，停損 $918 → 停損% = -1 × ($918 - $850) / $850 = **-8.0%** ✓（空單價格上漲是停損，仍為負值）
- 融券均價 $850，出場 $782 → 損益% = -1 × ($782 - $850) / $850 = **+8.0%** ✓（空單價格下跌為獲利，正值正確）

## 7. 批次鎖定機制（Lock Mechanism）

```
狀態：[[$782, 40%][x]] + [[$760, 30%][x]] + [[$750, 30%][🔒]]
                                                    ↑
                              自動計算 = 100% - 40% - 30% = 30%，readonly

[＋ 新增停損] 點擊後：
[[$782, 40%][x]] + [[$760, 30%][x]] + [[$750, 20%][x]] + [[新空白, 10%][🔒]]
                                               ↑                      ↑
                                      原最後一批解鎖            新最後一批鎖定
                                      10% = 100% - 40% - 30% - 20%

[x] 刪除[$760, 30%]後：
[[$782, 40%][x]] + [[$750, 20%][x]] + [[新空白, 40%][🔒]]
                                                    ↑
                                       40% = 100% - 40% - 20%
```

**規則總結：**
1. 最後一批 % 永遠 = $100 - \sum$ 前面所有批次 %（即時重算）
2. 最後一批只讀（`readonly` + 灰色背景 + 🔒 圖示）
3. 若前面批次輸入 % 合計 ≥ 100%，最後一批顯示 `0%` 並鎖定，不允許此狀態（需顯示警告）
4. 刪除批次後立即重算鎖定值并 re-render

## 8. 資料結構（localStorage）

```javascript
// localStorage key: 'rm-portfolio-rows'
[
  {
    id: 'row-uuid-1',                 // 唯一識別
    executed: true,                   // 已進場 checkbox
    direction: 'long',                // 'long' | 'short'
    ticker: 'NVDA',
    avgPrice: 850,
    capitalPct: 10,                   // 預計倉位比例 %（Input）
    stopLossBatches: [
      { price: 782, pct: 40 },        // 第1批（Input, Input）
      { price: 760, pct: 30 },        // 第2批（Input, Input）
      { price: 750, pct: null }       // 最後一批（Input, Output/locked）
    ],
    exitBatches: [
      { price: 950, pct: 50 },        // 第1批（Input, Input）
      { price: 1000, pct: null }      // 最後一批（Input, Output/locked）
    ]
  }
]
```

> `pct: null` 代表鎖定狀態，實際值由計算得出，不儲存在陣列中（每次 load 時重算）。

## 9. 計算觸發時機

| 觸發事件 | 需要重算的欄位 |
|:---|:---|
| 任何 Input 欄位修改（買入均價、倉位%、停損價、出場價）| 同一列所有 Output |
| 新增 / 刪除停損批次 | 同一列：平均停損%、建議股數、控風倉位%、R/R、帳戶貢獻 |
| 新增 / 刪除出場批次 | 同一列：平均損益%、R/R、損益金額、帳戶貢獻、狀態（自動結案判定）|
| 已進場 checkbox 切換 | 同一列：狀態、帳戶貢獻標籤、實際出場欄灰化狀態 |
| 方向切換（多/空）| 同一列所有 Output（含符號反轉）|
| `params_block` 更新 Capital / MaxRisk% | **所有列**全部重算 |

## 10. 概覽卡片（Overview Block）連動

`overview_block.js` 需彙總所有列的値：

| 概覽卡片 | 彙總公式 |
|:---|:---|
| 帳戶總風險 | $\sum$ 各列帳戶貢獻%（依狀態：規劃 → 停損貢獻；持倉/已結案 → 損益貢獻）|
| 全失敗停損總金額 | $\sum$ 各列（平均停損% × 控風倉位% × Capital）|
| 停損後剩餘資金 | Capital + 全失敗停損總金額（注意停損金額為負值）|

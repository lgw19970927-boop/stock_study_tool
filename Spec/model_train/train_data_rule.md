# YOLO 自定義 K 線型態：訓練資料與精準座標校正指南

本指南接續前述的系統架構，深入探討訓練資料的準備策略、YOLO 與 Python 的分工原理，以及如何解決圖片邊界留白（Padding）造成的 K 線數量換算誤差。

---

## 壹、 訓練資料筆數建議

針對金融圖表雜訊較多的特性，建議分階段準備訓練資料：

1. **初期驗證 (Proof of Concept)：300 ~ 500 個標記框**
   * 注意單位是「框（型態）」，若一張大圖中有兩個型態則算兩個框。
   * 此數量足以讓 YOLO 初步掌握特徵，用於串接系統測試邏輯是否暢通。
2. **加入背景圖 (Background Images)：50 ~ 100 張**
   * 準備完全沒有目標型態的 150 根 K 線大圖，且**不畫任何標記框**直接送入訓練。
   * **極度重要**：這能教導模型認識「無意義的橫盤與雜訊」，大幅降低實戰中的誤報率 (False Positives)。
3. **穩定上線標準：1000 個以上的標記框**
   * 涵蓋多頭、空頭、高波動與低波動市場的型態，確保模型具備高度穩定性。

## 貳、 破除迷思：為何絕對不能依 K 線數量分資料夾？

傳統程式思維會想將「10根的箱型」與「50根的箱型」分開存放，但在 YOLO 的世界裡，這反而會破壞模型的泛化能力。

### 核心原理解析：小孩與大人的分工
* **YOLO (只看形狀的小孩)**：模型不認識「時間」，只認得像素邊緣。如果你將各種長度（窄的、寬的、扁的）箱型混在一起教它，它會學到形狀的本質：「只要上下平齊就是在盤整」。
* **Python (拿皮尺的大人)**：系統（`service.py`）才是負責把關長度的人。它拿到 YOLO 找出的座標後，利用比例換算真實的 K 線數，並與前端設定的 `min_bars` / `max_bars` 進行比對過濾。

**結論**：將所有長度的型態全部混在同一個 `images/train/` 資料夾中訓練，讓 YOLO 專心找形狀，讓 Python 專心算長度。

---

## 參、 系統運作邏輯：先切分，再辨識

目前的系統 (`service.py`) 並非一次把所有歷史資料丟給 AI，而是嚴格執行以下流程：
1. **滾動切分**：以固定視窗（例如 150 根 K 線）在歷史數據上滾動，切出一小段一小段的 `chunk`。
2. **生成圖片**：將這個 `chunk` 送給 `chart_generator.py` 畫成固定比例的圖片。
3. **模型推理**：將畫好的圖片送進 YOLO 模型尋找型態座標。
4. **數學過濾**：換算座標對應的 K 線數量，剔除不符條件的訊號。

## 肆、 座標換算誤差與精準校正方案

### 1. 致命的誤差來源：圖片邊界的「留白 (Padding)」
系統預設的換算公式為 `(X座標 / 圖片總寬) * 150根`。
但如果畫圖套件 (`mplfinance`) 或看盤軟體在圖片左右兩側留了 10% 的空白，導致 150 根 K 線全擠在中間的 80% 區域，這個線性公式就會產生嚴重誤差（例如實際 48 根的型態，會被錯誤計算成 38 根而遭系統剔除）。

### 2. 解決方案 A：系統自產圖訓練法 (Zero Padding) —— 🌟 最精準推薦
從根本上消除邊距，讓「第 1 根 K 線貼死像素 0，第 150 根 K 線貼死圖片最右側」。
* **實作方式**：
  1. 修改 `utils/chart_generator.py`，將左右邊距設為 0：
     `scale_padding=dict(left=0.0, right=0.0, top=0.1, bottom=0.1)`
  2. 寫一支 Python 腳本，利用系統自己的 `chart_generator.py` 自動大量生成 150 根 K 線的圖片，並直接用這些無邊距圖片進行標註與訓練。
* **優勢**：訓練圖與實戰圖 100% 完美一致，座標換算達到絕對零誤差。

### 3. 解決方案 A 變體：TradingView 極限切齊截圖法 (手動 Zero Padding)
若想擁有 TradingView 的高畫質，且不打算修改 Python 的換算公式，則必須在截圖時實行「外科手術級的手動切齊」。
* **嚴格截圖規範**：
  1. **隱藏雜訊與座標軸**：關閉成交量、均線，且**絕對不可拍到右側的價格 Y 軸**。
  2. **固定 K 線數量**：畫面必須剛好呈現約 150 根 K 線。
  3. **極限拉框 (無呼吸空間)**：改用「手動拉框」截圖。截圖框的最左邊界必須剛好切過第 1 根 K 線的邊緣；最右邊界必須剛好切過最後 1 根 K 線的邊緣，不能留有一絲純黑的背景留白。
* **系統同步設定**：
  訓練圖既已手動緊貼邊緣，實戰推論產生的圖也必須緊貼邊緣。請務必將 `utils/chart_generator.py` 的邊距設為 0：
  `scale_padding=dict(left=0.0, right=0.0, top=0.1, bottom=0.1)`

### 4. 解決方案 B：TradingView 適應法 (Offset Calibration)
若想維持 TradingView 原本帶有留白的舒服視覺，則必須讓 Python 系統適應 TradingView 的留白比例。
* **截圖規範**：
  1. 保持固定長寬比（如 4:3 皆可）。
  2. 畫面固定顯示約 150 根 K 線，且**絕對不可拍到右側價格 Y 軸**。
* **程式碼校正**：
  測量 TradingView 的左右留白比例（例如左邊留白 5%，實際 K 線佔 80%），並將此常數寫入系統中：
  1. **同步畫圖設定 (`chart_generator.py`)**：
     `scale_padding=dict(left=0.05, right=0.15, top=0.1, bottom=0.1)`
  2. **加入偏移量換算 (`service.py`)**：
     ```python
     LEFT_PADDING_RATIO = 0.05
     EFFECTIVE_WIDTH_RATIO = 0.80
     
     # 扣除左邊留白像素，僅用有畫 K 線的有效寬度來計算比例
     effective_x1 = x1 - (img_width * LEFT_PADDING_RATIO)
     effective_x2 = x2 - (img_width * LEFT_PADDING_RATIO)
     effective_width = img_width * EFFECTIVE_WIDTH_RATIO
     
     local_start = int(max(0, (effective_x1 / effective_width) * chunk_len))
     local_end   = int(min(chunk_len, (effective_x2 / effective_width) * chunk_len))
     ```

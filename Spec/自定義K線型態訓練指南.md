# YOLO 自定義 K 線型態辨識系統：訓練與實作指南

本文件說明如何針對現有的 Python 量化回測系統，自行訓練並整合 YOLO 物件偵測模型，以精準辨識自定義的 K 線型態（如特定長度的箱型、盤整區等）。

## 壹、 系統現有架構解析 (基於 `service.py`)

目前的系統並非將 YOLO 當作單純的「影像分類器」，而是採用 **「大視窗滾動掃描 ＋ 程式後處理過濾」** 的高階架構。

### 1. 雙軌辨識機制

* **YOLO 視覺辨識**：負責辨識具備強烈視覺特徵的經典型態（如 W底、頭肩頂、三角收斂）。
* **規則法辨識**：針對純粹的「盤整區」，透過價格振幅（Sensitivity）、線性迴歸斜率與上下界觸碰測試進行數學篩選。

### 2. YOLO 模型運作邏輯

1. **大圖生成**：系統以固定大視窗（例如 150 根 K 線，或 `max_bars + 20`）進行滾動切片，透過 `chart_generator.py` 繪製成固定尺寸（如 640x640 或 800x600）的圖片。
2. **模型推論**：YOLO 模型在包含完整「上下文（前後趨勢）」的大圖中找出型態，並回傳邊界框的相對座標。
3. **K 線換算與過濾**：系統利用圖片寬度比例，將邊界框的 X 軸座標（`x1`, `x2`）換算回真實的 K 線 Index。若換算出的 K 線數量未落在前端設定的 `min_bars` 到 `max_bars` 區間內，則該訊號會被視為無效並剔除。

## 貳、 訓練資料準備核心原則：「外寬內緊」

為確保自行訓練的模型能完美對接現有系統的過濾邏輯，準備訓練資料時必須嚴格遵守以下原則：

### 1. 圖片擷取範圍 (Image Canvas)：提供上下文

* **作法**：訓練用的 K 線截圖必須是一張包含較大範圍（例如 150 根 K 線）的大圖。
* **目的**：讓模型在訓練時能看見型態發生前的趨勢與發生後的走勢（上下文）。這能大幅降低模型在毫無意義的橫盤中發出假訊號（False Positive）的機率。

### 2. 標註範圍 (Bounding Box)：極度精準

* **作法**：使用 LabelImg 等標註工具時，**標記框只能緊緊包住符合定義的那一小段 K 線區間**。
* **禁忌**：絕對不能將前方的上漲/下跌趨勢，或是後方的突破 K 線畫進框框內。
* **原因**：YOLO 產生的 `.txt` 標籤檔記錄的是「框框佔整張圖片的寬高比例」。系統會依靠這個比例來計算型態涵蓋了幾根 K 線。若標註範圍過大，系統在計算 K 線長度時會超出 `max_bars` 門檻，導致正確的訊號被系統邏輯誤殺。

### 3. 資料夾結構：混合訓練，不依 K 線數分類

* **作法**：各種不同長度（如 10 根、20 根、50 根）的同類型態截圖，必須全部混在同一個資料夾中進行訓練，絕對不要依照「K 線數量」或「時間長短」分資料夾。
* **原因**：YOLO 模型看的是像素特徵與形狀，並不認識「時間」。混合訓練能讓模型學到「不管長短，只要具備該形狀特徵就是目標型態」，大幅提升模型的泛化能力。實戰中的長度篩選，將全權交由 Python 後端依據前端拉桿（min_bars / max_bars）動態處理。

## 參、 截圖工具與尺寸設定

雖然 YOLO 模型在內部會自動縮放圖片，但為保證特徵（特別是 K 線的長寬比例與粗細）不變形，截圖時應盡量保持與系統 `chart_generator.py` 產生的圖片比例一致。

### 推薦截圖工具與實作方式

強烈建議使用可**「鎖定固定解析度 / 長寬比」**的工具進行大量截圖：

1. **ShareX (首選)**：
* 免費開源，可於設定中綁定快捷鍵為「擷取固定矩形區域」（例如鎖死 800x600 像素）。
* 按下快捷鍵後只需點擊滑鼠左鍵，即可瞬間完成等比例截圖並自動存檔，極大化資料收集效率。


2. **PicPick**：
* 內建「固定區域 (Fixed Region)」截圖功能，輸入指定的寬高數字後，利用出現的紅色固定框罩住看盤軟體畫面進行截圖。


3. **Lightshot**：
* 輕量級工具，拖曳時游標旁會顯示當前像素大小，需手動微調至目標尺寸。

## 肆、 自定義模型開發與整合 SOP

要將全新的自定義型態加入系統，請依序執行以下四個步驟：

### Step 1: 收集資料 (截圖)

開啟看盤軟體，將視窗調整至合適比例，利用截圖工具截取大量包含約 150 根 K 線的走勢圖。確保資料集中包含：

* 帶有標準自定義型態的完整走勢。
* 完全沒有該型態的正常走勢圖（Background Images，用於降低誤報率）。

### Step 2: 標記特徵 (Labeling)

將圖片匯入標註軟體（如 LabelImg）。針對圖中符合條件的型態進行精準框選，產生對應的 YOLO 格式 `.txt` 標籤檔。背景圖則直接略過不畫框。

### Step 3: 模型訓練

將準備好的「大圖 + txt 標籤檔」資料集送入 YOLO（如 YOLOv8）進行訓練。模型會自動從 txt 檔的相對座標中，學習在複雜的走勢背景裡尋找特定形狀。訓練完成後將取得 `.pt` 權重檔（例如 `custom_pattern.pt`）。

### Step 4: 系統整合與參數註冊

1. 將產出的 `.pt` 檔案放入專案的 `models` 資料夾中。
2. 開啟 `service.py`，將載入的模型路徑更改為你的新檔案。
3. 開啟 `pattern_mapping.py`，將你訓練時使用的 Class Name 與前端對應的參數註冊至字典中：

```python
YOLO_TO_FRONTEND: dict[str, Optional[str]] = {
    "Your_Custom_Class_Name": "frontend_checkbox_value",
    # ... 保留其他設定
}

FRONTEND_DISPLAY_NAMES: dict[str, str] = {
    "frontend_checkbox_value": "自定義型態名稱",
    # ... 保留其他設定
}

```

完成後，系統即可利用新模型進行視覺掃描，並透過前端設定的 K 線數量拉桿（`min_bars` / `max_bars`）精準過濾出你要的自定義型態。

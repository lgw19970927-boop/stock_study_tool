【儲存格內部 UI 細節放大看】
為了讓畫面看起來不雜亂，前端實作上必須將輸入框設計得非常緊湊。以下以「停損設定」欄位為例，放大其單元格內部的 UI 結構供你參考：

【欄位屬性 (Input/Output) 與核心計算邏輯】
開發時請務必實作以下全局參數與單列計算邏輯。
全局參數 (Global Variables)：由風險參數設定提供
- 初始資金 (Capital): [Input] 全局設定
- 單筆風險停損上限% (Max Risk Per Trade): [Input] 全局設定

各列欄位屬性與公式 (Row-level Formulas)：
1. 【持股 (Ticker)】: [Input]
2. 【進場規劃】:
   - 買入均價 (Avg Price): [Input]
   - 預計倉位比例 (Capital %): [Input] 預計購買倉位佔初始資金之比例。
   - 建議股數 (Suggested Shares): [Output]
     公式 = `INT( (預計倉位比例 * 初始資金) * IF(ABS(平均停損%) < 單筆風險停損上限%, 單筆風險停損上限% / ABS(平均停損%), 1) / 買入均價 )`
     說明：無條件捨去取整數，因為只能買整股。
   - 控風倉位% (Risk-Controlled Capital %): [Output]
     公式 = `(建議股數 * 買入均價) / 初始資金`
3. 【停損設定】 (支援動態 N 批次):
   - 停損價位 (Stop Loss Price): [Input]
   - 停損倉位佔比 (Stop Loss %): [Input] (註：該陣列最後一筆為 [Output]，自動鎖定為 `100% - 前面批次加總`)
   - 平均停損% (Avg Stop Loss %): [Output]
     公式 = `Σ [ ((各批停損價 - 買入均價) / 買入均價) * 該批停損倉位佔比% ]`
4. 【實際出場】 (支援動態 N 批次):
   - 平倉出場價位 (Exit Price): [Input]
   - 平倉出場倉位佔比 (Exit %): [Input] (註：該陣列最後一筆為 [Output]，自動鎖定為 `100% - 前面批次加總`)
   - 平均損益% (Avg P/L %): [Output]
     公式 = `Σ [ ((各批出場價 - 買入均價) / 買入均價) * 該批出場倉位佔比% ]`
5. 【交易績效】:
   - 盈虧比 (R/R Ratio): [Output]
     公式 = `平均損益% / ABS(平均停損%)`
   - 實際損益金額 (Actual P/L Amount): [Output]
     公式 = `平均損益% * 建議股數 * 買入均價`
6. 【帳戶貢獻】:
   - 潛在風險 / 實際獲利 (Portfolio Risk/Reward Contribution): [Output]
     公式 = `IF(尚無實際平倉出場數據, 平均停損% * 控風倉位%, 平均損益% * 控風倉位%)`

【核心互動與實作規格】
1. 單元格內垂直擴充 (Dynamic Rows in Cell)：
   - 在【停損設定】與【實際出場】欄位中，允許使用者點擊「➕ 新增」按鈕。
   - 點擊後，只在該單元格「內部」往下新增一列「價位 @ 比例」的輸入框 (Input)，表格的其他欄位需保持垂直置中對齊 (Vertical-align: middle)，不破壞整體版面。
   - 單元格內部排版邏輯為：上半部是輸入區 (Input)，使用水平線 (---) 隔開後，下半部是即時計算的結果區 (Output，如平均停損)。

2. 自動防呆與比例鎖定機制 (Lock Mechanism)：
   - 無論新增幾個批次，該欄位最後一筆的佔比 % 數必須由系統「自動計算」(`100% - 前面所有批次加總`)。
   - 最後一筆的 % 數輸入框必須是唯讀/鎖定狀態 (Disabled / Readonly)，確保總比例永遠等於 100%。
   - 除了最後一筆，其他批次旁邊都要有 [✖] 刪除按鈕。

3. 狀態連動顯示 (Status UI)：
   - 當狀態為「規劃」時：【實際出場】欄位應呈現反灰或顯示「等待出場」；【帳戶貢獻】顯示橘/紅色的「預估風險」數值。
   - 當狀態為「持倉/已結案」時：【實際出場】輸入框解鎖；如果有輸入出場價位，【帳戶貢獻】切換顯示為綠/紅色的「實際獲利/虧損」數值。

4. 數據視覺化強化：
   - 「盈虧比 (R/R Ratio)」：若大於 1 顯示綠色，小於 1 顯示紅色。
   - 輸出的數值（如建議股數、平均停損、實際損益金額）字體請加粗，輸入框 (Input) 的字體維持常規，以引導視覺焦點。
